# AdaRePO: Adaptive Reference-guided Policy Optimization for Molecular Generation

*Method Formalization Draft — v16*

---

## 1. Problem Definition

**Task.**
Given a reference molecule $m_\text{ref}$ and a natural-language instruction $q$ specifying a property $p$ and a desired direction $d \in \{\texttt{increase}, \texttt{decrease}\}$, the agent must generate a novel molecule $m'$ that (i) improves property $p$ in direction $d$ relative to $m_\text{ref}$, while (ii) maintaining structural similarity to $m_\text{ref}$.

**State.**
Each training instance is a tuple $(q, m_\text{ref})$. The prompt $x$ is constructed by prepending a fixed system prompt $s$ and the user instruction $q$:
$$x = [s ; q]$$
The model receives $x$ as tokenized input.

**Action.**
An action $a$ is a complete text sequence generated by the policy $\pi_\theta$, structured as:
$$a = \texttt{<think>} \; \tau \; \texttt{</think>} \; \texttt{<answer>} \; m' \; \texttt{</answer>}$$
where $\tau$ is a chain-of-thought reasoning trace and $m'$ is a SMILES string.

**Trajectory.**
For each prompt $x$, the policy generates $G$ independent completions $\{a_1, \dots, a_G\}$ via sampling (in this work, $G = 4$). Each $(x, a_j)$ pair constitutes a trajectory. One training step processes $B$ prompt groups, yielding $B \times G$ trajectories total.

**Reward.**
Each trajectory is evaluated by a composite reward function:
$$r(a_j \mid q, m_\text{ref}) = w_\text{sim} \cdot \text{Sim}(m'_j, m_\text{ref}) + w_\text{prop} \cdot \Delta_p(m'_j, m_\text{ref}, d)$$
where:
- $\text{Sim}(m', m_\text{ref}) = \text{Tanimoto}(\text{Morgan}(m'), \text{Morgan}(m_\text{ref}))$ is the Morgan fingerprint Tanimoto similarity (radius 2, 2048 bits);
- $\Delta_p(m', m_\text{ref}, d) = \begin{cases} p(m') - p(m_\text{ref}) & \text{if } d = \texttt{increase} \\ p(m_\text{ref}) - p(m') & \text{if } d = \texttt{decrease} \end{cases}$ is the signed property improvement;
- $w_\text{sim}, w_\text{prop}$ are configurable weights (default: $0.5, 0.5$).

An action is assigned $r = 0$ if $m'$ is chemically invalid or $\text{Sim}(m', m_\text{ref}) < \tau_\text{sim}$ (a minimum similarity gate; default $\tau_\text{sim} = 0.2$).

The **reference reward** $v_\text{ref}(q)$ is the reward obtained by echoing $m_\text{ref}$ as the answer. By construction, $\text{Sim}(m_\text{ref}, m_\text{ref}) = 1$ and $\Delta_p = 0$, so $v_\text{ref}(q) = w_\text{sim}$.

---

## 2. Repository Definition

**Molecule Memory Bank $\mathcal{B}$.**
$\mathcal{B}$ is a dictionary indexed by query hash $h(q)$. Each entry stores up to $K$ high-value molecules:
$$\mathcal{B}(q) = \{(m_i, r_i, t_i)\}_{i=1}^{|\mathcal{B}(q)|}, \quad |\mathcal{B}(q)| \leq K$$
where $m_i$ is a SMILES string, $r_i$ is the reward at the time of promotion, and $t_i$ is the step at which the entry was last updated.

**Unit of storage.**
Each entry is a single molecule—not a full trajectory. The memory bank does not store token sequences, log-probabilities, or advantages; it stores only the SMILES string and scalar metadata.

**Eviction.**
Entries older than $T_\text{max}$ steps are evicted:
$$\mathcal{B}(q) \leftarrow \{(m, r, t) \in \mathcal{B}(q) \mid t_\text{current} - t \leq T_\text{max}\}$$

---

## 3. Active Selection Mechanism

### 3.1 Promotion Rule (Memory Bank)

After reward computation at step $n$, each generated molecule $m'_j$ from query $q$ is evaluated for promotion into $\mathcal{B}(q)$. The promotion criterion is:

$$\text{Promote}(m'_j) = \mathbf{1}\Big[ r(m'_j) > v_\text{ref}(q) + \delta \;\;\wedge\;\; \text{Sim}(m'_j, m_\text{ref}) \geq \tau_\text{promote} \;\;\wedge\;\; \text{Valid}(m'_j) \Big]$$

where $\delta$ is the promotion margin (default 0.05) and $\tau_\text{promote}$ is the minimum Tanimoto similarity for promotion (default 0.2).

If $|\mathcal{B}(q)| = K$ and $\text{Promote}(m'_j) = 1$, the entry with lowest $r_i$ is replaced if $r(m'_j) > \min_i r_i$.

### 3.2 Active Reference Selection

At guidance-loss time, the training target for query $q$ is selected as:
$$m^*(q) = \operatorname{argmax}_{m \in \{m_\text{ref}\} \cup \mathcal{B}(q)} r(m)$$

When $m^*(q) \neq m_\text{ref}$, the guidance loss trains toward the model's own best historical generation. We call this **self-distillation**: the policy's past best output serves as the teacher.

## 4. Training Objective

The total loss for one mini-batch consists of three components:

### 4.1 RL Loss (GRPO Surrogate)

Group-normalized advantages are computed per prompt group:
$$\hat{A}_j^{(i)} = \frac{r(a_j^{(i)}) - \bar{r}^{(i)}}{\sigma_r^{(i)} + \epsilon}, \quad \bar{r}^{(i)} = \frac{1}{G}\sum_{j=1}^{G} r(a_j^{(i)}), \quad \sigma_r^{(i)} = \text{std}\{r(a_j^{(i)})\}_{j=1}^{G}$$

The per-token RL loss with KL regularization:
$$\mathcal{L}_\text{RL} = -\frac{\sum_{j,t} \big[\exp(\log \pi_\theta(a_{j,t} | x, a_{j,<t}) - \text{sg}[\log \pi_\theta(a_{j,t} | \cdot)]) \cdot \hat{A}_j - \beta_\text{KL} \cdot D_\text{KL}^{(j,t)}\big] \cdot c_{j,t}}{\sum_{j,t} c_{j,t}}$$

where $c_{j,t}$ is the completion mask, $\text{sg}[\cdot]$ is stop-gradient, and $D_\text{KL}^{(j,t)} = \exp(\log \pi_\text{ref} - \log \pi_\theta) - (\log \pi_\text{ref} - \log \pi_\theta) - 1$ is the per-token reverse KL divergence.

**Note:** The $\exp(\log \pi_\theta - \text{sg}[\log \pi_\theta])$ construction evaluates to $1$ at forward time but preserves the gradient $\nabla_\theta \log \pi_\theta$, implementing the REINFORCE-style policy gradient. This is the standard TRL/GRPO implementation.

### 4.2 Guidance Loss (Answer-Level Imitation)

A supervised loss on the selected reference molecule $m^*(q)$:
$$\mathcal{L}_\text{guide} = -\frac{\sum_t \log \pi_\theta(m^*_t | x, m^*_{<t}) \cdot s_t}{\sum_t s_t + \epsilon}$$

where $s_t$ is a mask that isolates the SMILES token span within the completion (constructed by `replace_smile`, which substitutes the generated SMILES with $m^*$ in the decoded completion text, preserving the reasoning trace).

### 4.3 Combined Objective

$$\mathcal{L} = \mathcal{L}_\text{RL} + \beta_\text{guide}(q) \cdot \mathcal{L}_\text{guide}$$

#### Dynamic Beta Controller

$\beta_\text{guide}$ is computed per prompt group (not per sample) via the **sigmoid-gap** schedule:

$$\beta_\text{guide}(q) = \beta_\min + (\beta_\max - \beta_\min) \cdot \sigma\big(-\alpha \cdot (v_\text{top}(q) - v_\text{ref}(q))\big)$$

where:
- $v_\text{top}(q) = \frac{1}{k}\sum_{j \in \text{top-}k} r(a_j^{(q)})$ is the mean of the top-$k$ generation rewards ($k = \lfloor G \cdot f_\text{top} \rfloor$, with $f_\text{top} = 0.33$);
- $\sigma(\cdot)$ is the logistic sigmoid;
- $\alpha$ controls the transition sharpness (default 3.0).

**Behavior:**
| Regime | Gap | $\beta_\text{guide}$ | Effect |
|--------|-----|---------------------|--------|
| Policy $\ll$ reference | $v_\text{top} \ll v_\text{ref}$ | $\to \beta_\max$ | Strong imitation (IL-dominant) |
| Policy $\approx$ reference | $v_\text{top} \approx v_\text{ref}$ | $\approx \frac{\beta_\max + \beta_\min}{2}$ | Balanced |
| Policy $\gg$ reference | $v_\text{top} \gg v_\text{ref}$ | $\to \beta_\min$ | RL-dominant |

**Note:** Unlike vanilla RePO where $\beta_\text{guide} = 1$ (implicit), the dynamic controller allows $\beta_\text{guide} > 1$ when the policy lags behind the reference, providing stronger-than-unit imitation drive.

## 5. Complete Training Procedure

**Input:** Dataset $\mathcal{D} = \{(q_i, m_{\text{ref},i})\}_{i=1}^{N}$, pre-trained LLM $\pi_\theta$, reference model $\pi_\text{ref}$ (frozen copy of $\pi_\theta$ at initialization or the base model under LoRA).

**Initialize:** Memory bank $\mathcal{B} \leftarrow \emptyset$; EMA sigma tracker $\hat{\sigma}(q) \leftarrow \emptyset$.

**For each training step $n = 1, \dots, N_\text{steps}$:**

1. **Sample mini-batch.** Draw $B$ prompt groups from $\mathcal{D}$ (standard dataloader, no active selection at the batch level).

2. **Sync policy to vLLM.** If $n$ differs from the last sync step, copy $\pi_\theta$'s weights to the vLLM inference engine.

3. **Adaptive temperature selection.** For each prompt $q$ in the batch:
   - Look up $\hat{\sigma}(q)$ from the EMA tracker.
   - If $\hat{\sigma}(q) < \tau_\sigma$ (default 0.05): set sampling temperature $T(q) = T_\text{high}$ (default 1.3).
   - Otherwise: $T(q) = T_\text{base}$ (default 0.9).

4. **Generate.** For each prompt $q$, sample $G$ completions $\{a_1, \dots, a_G\} \sim \pi_\theta(\cdot | x; T(q))$ using vLLM with per-prompt `SamplingParams`.

5. **Compute rewards.** Evaluate $r(a_j | q, m_\text{ref})$ for all $B \times G$ completions, and $v_\text{ref}(q)$ for each prompt.

6. **Compute advantages.** Group-normalize rewards within each prompt group (GRPO):
$$\hat{A}_j = \frac{r_j - \bar{r}}{\sigma_r + \epsilon}$$

7. **Update EMA sigma.** For each prompt $q$:
$$\hat{\sigma}(q) \leftarrow \alpha_\text{ema} \cdot \sigma_r(q) + (1 - \alpha_\text{ema}) \cdot \hat{\sigma}(q)$$

8. **Compute dynamic beta.** Apply the sigmoid-gap controller using $v_\text{top}(q)$ and $v_\text{ref}(q)$.

9. **Update memory bank.** For each completion $a_j$:
   - Extract $m'_j$ from the $\texttt{<answer>}$ tag.
   - If $\text{Promote}(m'_j) = 1$: insert/update $\mathcal{B}(q)$.

10. **Select active reference.** For each prompt $q$:
$$m^*(q) = \operatorname{argmax}_{m \in \{m_\text{ref}\} \cup \mathcal{B}(q)} r(m)$$

11. **Build solution tokens.** Replace the SMILES in each decoded completion with $m^*(q)$ via `replace_smile`, then tokenize and mask.

12. **Compute loss.** For each gradient accumulation sub-step:
$$\mathcal{L} = \mathcal{L}_\text{RL}(\pi_\theta, \hat{A}, \pi_\text{ref}) + \beta_\text{guide}(q) \cdot \mathcal{L}_\text{guide}(\pi_\theta, m^*)$$

13. **Update $\pi_\theta$.** Standard optimizer step (AdamW with cosine LR schedule, DeepSpeed ZeRO-3).

## 6. Essential Differences from Vanilla RePO

| Aspect | RePO | AdaRePO |
|--------|------|---------|
| **Guidance coefficient** | $\beta_\text{guide} = 1$ (fixed, implicit) | $\beta_\text{guide}(q) \in [\beta_\min, \beta_\max]$, adaptive per prompt group via sigmoid-gap on $v_\text{top} - v_\text{ref}$ |
| **Guidance target** | Always $m_\text{ref}$ (the dataset gold reference) | $m^*(q) = \operatorname{argmax}\{m_\text{ref}, \mathcal{B}(q)\}$; can be the model's own best historical generation (self-distillation) |
| **Sampling temperature** | Fixed $T$ for all prompts | Per-prompt adaptive $T(q)$: collapsed prompts ($\hat{\sigma}(q) < \tau_\sigma$) receive higher $T$ to restore generation diversity |
| **Per-prompt state** | Stateless across steps | EMA reward-sigma tracker $\hat{\sigma}(q)$ persists across steps; memory bank $\mathcal{B}(q)$ accumulates high-value molecules |

**In summary, the three concrete algorithmic additions are:**

1. **Dynamic $\beta_\text{guide}$:** The IL-vs-RL balance is query-adaptive. Prompts where the policy already surpasses the reference receive reduced imitation pressure ($\beta \to \beta_\min$); prompts where the policy lags receive amplified imitation ($\beta \to \beta_\max$, potentially $> 1$). This was present since v15.1.

2. **Memory bank self-distillation:** The guidance target is no longer static. When the policy has historically generated a molecule $m' \in \mathcal{B}(q)$ that surpasses $m_\text{ref}$, the guidance loss trains toward $m'$ instead. This creates a positive feedback loop: improvements in generation quality upgrade the teacher target, which in turn drives further improvement. New in v16.

3. **Adaptive sampling temperature:** Per-prompt EMA-tracked reward variance detects "collapsed" prompts (low diversity across $G$ completions). These receive elevated temperature to break mode collapse. New in v16.

## 7. Undetermined Items

The following aspects are not fully specified in the current codebase and require clarification or deliberate design decisions:

1. **Batch-level active selection is absent.** The dataloader samples uniformly from $\mathcal{D}$. There is no mechanism to preferentially sample "informative" prompts (high $\sigma$, near learning frontier) at the batch construction level. The experience buffer module (`ExperienceBuffer`) has been implemented but is not yet wired to replace batch slots — it currently only logs diagnostics. *Whether and how to integrate batch-level selection (e.g., replacing stable-good examples with buffer replays) remains an open design question.*

2. **Memory bank reward staleness.** Entries in $\mathcal{B}(q)$ store the reward $r_i$ computed at promotion time. The reward function is deterministic given $m'$ and $m_\text{ref}$, so the stored reward is not stale in the usual RL sense. However, *the promotion criterion compares against $v_\text{ref}(q)$ which is also deterministic*. This means once an entry is promoted, it is never re-evaluated — even if the reward function or its weighting were to change during training (currently they do not).

3. **Cross-epoch memory bank persistence.** $\mathcal{B}$ is not reset between epochs; it accumulates across all training steps. With $T_\text{max} = 500$ steps and $\sim 748$ total steps over 4 epochs, later-epoch entries can persist but early-epoch entries are evicted. *Whether resetting $\mathcal{B}$ at epoch boundaries would be beneficial is untested.*

4. **Interaction between self-distillation target and advantage.** When $m^*(q) \neq m_\text{ref}$, the guidance loss trains toward a higher-reward target, but the advantages $\hat{A}_j$ are still computed relative to the *generated* completions' mean reward — not relative to $m^*$. *There is no explicit coupling between the active reference selection and the RL advantage computation.*

5. **Adaptive temperature threshold sensitivity.** The threshold $\tau_\sigma = 0.05$ is a fixed hyperparameter. In practice, the v16 training run shows that by late training, *zero prompts* are classified as collapsed ($\hat{\sigma}(q) > 0.47$ for all prompts). This suggests the adaptive temperature mechanism may be primarily active in early training, and $\tau_\sigma$ may need task-specific tuning. *Whether this early-only activation is sufficient or indicates a too-conservative threshold is unknown.*

6. **$\beta_\text{guide}$ is averaged over the batch.** In `compute_loss`, $\beta_\text{guide}$ is reduced to a scalar via `.mean()` before multiplying $\mathcal{L}_\text{guide}$. This means per-prompt beta modulation is lost at the loss level — the guidance loss is weighted by the *batch-average* beta, not per-sample. *This is a potential discrepancy between the intended per-prompt design and the actual implementation.*

7. **Solution mask construction.** The guidance target tokenization relies on `replace_smile`, which substitutes the SMILES in the *decoded completion text* with $m^*$, then re-tokenizes. If the model's completion does not contain a valid SMILES in `<answer>` tags, the replacement fails silently (position defaults to $(0, 0)$, producing an all-zero mask). *The fraction of such failures and their impact on the guidance loss magnitude are not explicitly tracked.*

8. **Priority weighting (disabled).** The codebase contains a priority weighting module that multiplies advantages by an informativeness score (variance $\times$ frontier proximity). This is disabled in v16 (`use_priority_weighting: false`). *Its interaction with the other mechanisms and whether it is redundant given adaptive temperature is untested.*
